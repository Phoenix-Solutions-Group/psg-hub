# Field: RC_ImportBatchID

## Field Specification

| Property | Value |
|----------|-------|
| Table | Repair Customer |
| Field Name | RC_ImportBatchID |
| Type | Text |
| Auto-Entry | None |
| Index | All values |
| Validation | None |
| Storage | Stored (default) |
| Global | No |

## Purpose

Links Repair Customer records back to their import batch for recall operations. When the processing script creates a new RC record from a staging record, it copies IS_BatchID from Import_Staging to RC_ImportBatchID on the new Repair Customer record.

## Batch ID Format

| Property | Value |
|----------|-------|
| Format | Sequential: BATCH-0001, BATCH-0002, ... |
| Zero-Padded | 4 digits minimum |
| Generated By | Calling system (browser utility) before POSTing to Import_Staging |
| Uniqueness | One batch ID per import session (all records from one file upload) |

## Usage

- **Processing script (06-02):** Copies IS_BatchID from staging to RC_ImportBatchID when creating new RC record
- **Recall script (06-03):** Finds all RC records with a given RC_ImportBatchID and deletes them
- **Monitoring:** Query RC_ImportBatchID to see which records came from which import batch

## Index Requirement

The "All" index is required because the recall script performs a Find on this field across 281,000+ records. Without an index, the Find would be unacceptably slow.

## Impact Assessment

| Concern | Assessment |
|---------|------------|
| Existing records affected | No. Adding a text field with no auto-entry means all 281,077 existing records will have this field empty. This is correct -- they were not API-imported. |
| Schema rebuild required | No. Adding a new field does not trigger a schema rebuild. |
| Existing calculations affected | No. No existing calculation references RC_ImportBatchID. |
| File size impact | Negligible. Empty text field adds near-zero storage. Index builds incrementally as records are created via API. |
| Downtime required | No. Field can be added while the database is live and users are connected. |
| Rollback plan | Delete the field. No other schema elements depend on it. |

## Relationship to Import_Staging

| Import_Staging Field | Repair Customer Field | Set By |
|---------------------|----------------------|--------|
| IS_BatchID | RC_ImportBatchID | Processing script copies value during record creation |

No formal FileMaker relationship is needed. The processing script reads IS_BatchID from the staging record's variables and writes it to RC_ImportBatchID on the new RC record via Set Field.
