<objective>
## Goal
Extract PSG brand tokens from the design system zip + brand guidelines URL via `/brandkit`, codify them as `tokens/psg.json`, rebuild the Tailwind v4 theme from those tokens (CSS vars + theme.extend), install shadcn/ui scaffolding configured to consume the PSG vars (not Slate defaults), and replace the placeholder page with a token-driven typography + button sample that proves the theme actually flows through.

## Purpose
Anti-AI-slop pillar #2 (brand tokens before components) is binding. Every subsequent UI plan inherits this theme. If the tokens aren't right, every page that follows will leak the default shadcn look. This plan locks the visual foundation.

## Output
- `tokens/psg.json` — canonical PSG palette, type scale, spacing, radii, shadows, motion durations
- `tokens/psg-source-notes.md` — short audit log of where each token came from (zip file path, guidelines section, judgment calls)
- Tailwind v4 theme + `globals.css` consuming the tokens as CSS vars
- shadcn/ui installed with PSG-themed `components.json`; Button + Card primitives generated and re-themed (no raw shadcn defaults)
- `app/page.tsx` replaced with a "brand identity proof" page that renders the type scale, color swatches, and a Button — visibly PSG, not generic
- `/impeccable critique` exit gate passed (binding)
</objective>

<task type="auto">
  <name>Task 2: Rebuild Tailwind v4 theme from tokens</name>
  <files>app/globals.css, tailwind.config.ts</files>
  <action>
    Tailwind v4 supports `@theme` in CSS. Use that pattern:

    1. In `app/globals.css`:
       - `@import "tailwindcss";`
       - `@theme { ... }` block: map every token from psg.json to a CSS var (e.g., `--color-primary: oklch(...);`, `--font-display: "<family>", sans-serif;`)
       - `:root { ... }` block for any vars used directly (e.g., for shadcn defaults that read `--background`, `--foreground`)
       - No raw hex values in globals.css — every color references a token value (paste from psg.json)

    2. If tailwind.config.ts is needed (Tailwind v4 minimizes config), keep it minimal:
       - content: scan app/**, components/**, lib/**
       - plugins: none yet
       - DO NOT override theme via JS — `@theme` in CSS is the source

    3. Add a font loader if the PSG type family is a webfont:
       - If guidelines specify a Google Font: import via `next/font` in app/layout.tsx, expose as a CSS var
       - If guidelines specify a licensed font: document in source notes, leave a TODO comment in globals.css pointing at the licensing decision

    Avoid:
    - Overriding `@theme` with `tailwind.config.ts` JS — pick ONE source of truth
    - Setting any color outside the token system
    - Loading more than the 2-3 weights actually used (perf)
  </action>
  <verify>
    pnpm build
    # Confirm the theme block exists
    grep -q '@theme' app/globals.css
    grep -q '@import "tailwindcss"' app/globals.css
  </verify>
  <done>AC-2 satisfied: build succeeds; theme defined from tokens via @theme; no raw hex outside tokens/.</done>
</task>
