<objective>
## Goal
Scaffold the Next.js customer dashboard with Supabase Auth, multi-tenant PostgreSQL schema with RLS, and PSG brand design tokens. After this plan, users can sign up, log in, and land on an authenticated dashboard shell with proper tenant isolation.

## Purpose
This is the foundation everything else in Phase 4 builds on. Auth, multi-tenancy, and the design system must be solid before adding features.

## Output
- Next.js 15 app with App Router in dashboard/ directory
- Supabase Auth (email/password) with login/signup pages
- Multi-tenant PostgreSQL schema with RLS policies
- PSG brand design tokens (Authority Palette + Clarity Teal)
- shadcn/ui base components styled to brand
- Auth middleware protecting dashboard routes
</objective>

<task type="auto">
  <name>Task 3: Multi-tenant PostgreSQL schema with RLS</name>
  <files>supabase/migrations/001_initial_schema.sql</files>
  <action>
    1. Create supabase/migrations/ directory
    2. Write initial migration with tables:
       - profiles (id uuid PK refs auth.users, email, full_name, role, created_at)
       - shops (id uuid PK, name, slug, website_url, phone, city, state, created_at, updated_at)
       - shop_members (id uuid PK, shop_id refs shops, profile_id refs profiles, role enum(owner,manager,viewer), created_at)
       - content_items (id uuid PK, shop_id refs shops, title, content_type, status enum(draft,pending_review,approved,published,rejected), body text, meta jsonb, created_by uuid, created_at, updated_at)
       - agent_runs (id uuid PK, shop_id refs shops, agent_type text, status text, started_at, completed_at, result jsonb)
    3. Add RLS policies:
       - shops: users can SELECT only shops they are members of
       - shop_members: users can SELECT only their own memberships
       - content_items: users can SELECT/INSERT/UPDATE only for their shops
       - agent_runs: users can SELECT only for their shops
    4. Create function: get_user_shop_ids() returns shop IDs for current auth user
    5. Add trigger: on auth.users insert, create profiles row
    6. Add indexes on foreign keys and commonly queried columns
  </action>
  <verify>cat supabase/migrations/001_initial_schema.sql | head -20</verify>
  <done>AC-3 satisfied: Schema defines multi-tenant tables with RLS policies enforcing tenant isolation</done>
