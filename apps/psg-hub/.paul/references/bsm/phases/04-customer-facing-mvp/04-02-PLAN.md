<objective>
## Goal
Build the core dashboard feature views: content management table, content detail with inline preview, agent status cards, shop settings, and the dual approval workflow API. After this plan, shop owners can view content, preview it, approve or reject it, and see agent activity.

## Output
- Content list page with status filtering
- Content detail page with embedded preview and approval buttons
- Agent runs page showing recent activity
- Settings page with shop profile
- Approval API routes (approve/reject)
</objective>

<acceptance_criteria>

## AC-1: Content management views
```gherkin
Given a user is authenticated and has shop membership
When they visit /dashboard/content
Then they see a table of content items with title, type, status, and date
And they can click a content item to view its detail page
```

## AC-2: Content preview and approval
```gherkin
Given a user views a content detail page
When the content has status pending_review
Then they see the content body rendered with formatting
And they see Approve and Reject buttons
And clicking Approve changes status to approved
And clicking Reject changes status to rejected
```

## AC-3: Agent activity and settings
```gherkin
Given a user visits /dashboard/agents
Then they see cards for each agent type with last run status
Given a user visits /dashboard/settings
Then they see their shop profile information
```

    - If status is pending_review: show ApprovalActions component
    - ApprovalActions has Approve (green) and Reject (red) buttons
    - Buttons call server actions that update status
