<objective>
## Goal
Build a UI/UX designer agent skill that automatically crawls a shop's website, extracts the complete site shell and page-type layouts, saves reusable HTML templates, and integrates into the agent swarm so every content preview is pixel-accurate from day one.

## Purpose
Currently, template extraction requires manual effort: someone must save a page's HTML source, then run extract-template.js. The UI/UX designer agent eliminates that manual step. When a new shop is onboarded, this agent runs first as part of the swarm, crawling the site and building templates before the content writer produces anything. Every subsequent content preview automatically uses the real site HTML.

## Output
- New Claude Code skill: `bsm-site-designer`
- Updated `paperclip.config.json` with 5th agent registered
- Updated `preview/extract-template.js` with multi-page support
- Updated `preview/generate-preview.js` with page-type template selection
- Tracy's templates validated with multiple page types
</objective>

<task type="auto">
  <name>Task 1: Build bsm-site-designer agent skill</name>
  <files>~/.claude/skills/bsm-site-designer/SKILL.md</files>
  <action>
    Create a new Claude Code skill following the same pattern as bsm-content-writer.

    The skill takes as input:
    - shop_slug: identifier for the shop (e.g., tracys-collision-center)
    - website_url: the shop's website root URL
    - pages_to_crawl: optional list of specific URLs (auto-discovers if omitted)

    The skill workflow:
    1. Crawl the site using Firecrawl MCP to get full HTML of key pages:
       - Home page (/)
       - About page (auto-detect: /about, /about-us, /about-tracys, etc.)
       - A service page (auto-detect from nav links)
       - A blog post page (auto-detect from /blog, /news, etc.)
       - Contact page (auto-detect)
    2. For each crawled page, save the full HTML to shops/{shop_slug}/content/{page-type}.html
    3. Run the extraction logic (same as extract-template.js) on the primary page to get the site shell
    4. For each page type, extract the content area layout structure (the HTML between header and footer) to identify how that page type wraps content (e.g., full-width vs sidebar, Elementor sections, etc.)
    5. Save page-type layout metadata to preview/templates/{shop_slug}/layouts.json:
       ```json
       {
         "default": "blog",
         "types": {
           "blog_post": { "source": "blog.html", "content_wrapper": "<div class='...'>", "has_sidebar": false },
           "service_page": { "source": "service.html", "content_wrapper": "<section class='...'>", "has_sidebar": false },
           "about": { "source": "about.html", "content_wrapper": "..." }
         }
       }
       ```
    6. Output a summary of what was extracted: pages crawled, fragments saved, page types identified

    The skill persona: "Site designer agent that replicates existing websites as templates for content preview. Does not redesign or improve sites. Faithful extraction only."

    Collision repair site patterns to handle:
    - Elementor/OceanWP (Tracy's pattern)
    - WordPress themes (Divi, Avada, Astra)
    - Squarespace, Wix (simplified extraction)
    - Static HTML sites

    Include inter-agent interface:
    - Triggered by: shop onboarding (runs first before any content agent)
    - Outputs to: content-writer (templates ready signal)
    - Can be re-invoked manually to refresh templates
  </action>
  <verify>Skill file exists at ~/.claude/skills/bsm-site-designer/SKILL.md with all sections populated</verify>
  <done>AC-1 partially satisfied: skill definition complete with crawl + extract + save workflow</done>
</task>

    - If no matching page type, fall back to the default (blog) layout
    - Keep the current behavior as the fallback when no layouts.json exists
