<objective>
## Goal
Plug the campaigns UI into the /ads page: list campaigns with metrics, create from template, pause/enable/edit/delete per campaign, and manually sync metrics. Consumes 05-03 routes; no new endpoints.

## Purpose
Without this plan, Phase 5 ships with working plumbing and a blank-looking ads page. Shop owners see their account linked but cannot actually run campaigns. This plan is the final slice of Phase 5 — after this, BSM delivers the paid-search value Performance tier promises.

## Output
- CampaignsSection below AccountsTable when a linked account exists
- CampaignsTable w/ metrics, keyboard-accessible rows, soft-cap footer
- CreateCampaignModal w/ template picker + preflight-aware budget input + precision guard
- CampaignDetailModal mirroring 05-02 a11y + stale-data hint + Refresh
- MetricsSummary (4 plain cards; tolerant of empty/missing metrics)
- SyncButton w/ aria-live messaging + campaign-name error mapping
- Client-safe template subset w/ drift-detection test
- focus-trap pure helper + test
- Tests covering validation, formatting, drift, focus-trap, metrics tolerance
</objective>

<acceptance_criteria>

## AC-1: CampaignsSection renders when account linked
```gherkin
Given a Performance-tier shop with ≥1 linked Google Ads account
When the user views /ads
Then below the AccountsTable, a CampaignsSection renders with the shop's campaigns
And if zero campaigns, an empty state shows "No campaigns yet" + a "Create your first campaign" button (owners/managers only)
And if no linked accounts, CampaignsSection does not render
```

## AC-2: Create campaign from template
```gherkin
Given the CreateCampaignModal is open
When the user picks a template
Then the modal shows template metadata (name, description, keyword counts, headline/description counts, default daily_budget_micros)
And the user can adjust daily_budget_micros via a dollar input (handled via dollarsToMicros helper — 2-decimal precision enforced; non-finite/≤0 rejected)
And the user can edit the name (pre-filled "<Shop name> — <Template name>"); client validates empty + >255 chars + budget cap
And on submit, POST /api/ads/google/campaigns w/ { shop_id, template_id, daily_budget_micros, name }
And 400 responses surface `missing` array in plain language
And other non-200 responses surface server error verbatim
And on success, modal closes + router.refresh()
```

## AC-3: Campaign detail modal — pause/enable/budget/delete + stale hint
```gherkin
Given the user opens a campaign row
When the detail modal renders
Then it shows: name, status, current budget, last metrics snapshot, "Loaded N seconds ago" hint + Refresh button
And controls gated via selectCampaignControls(role, status):
  - Pause (owner|manager, status='enabled') → PUT { status: 'paused' }
  - Enable (owner only, status='paused') → PUT { status: 'enabled' }
  - Budget edit (owner|manager, status in ('paused','enabled')) → client ±50% delta + cap pre-validation → PUT
  - Delete (owner only) → window.confirm with copy clarifying "REMOVED in Google Ads, no longer appears in BSM, cannot be undone" → DELETE
And modal a11y: role="dialog", aria-modal, aria-labelledby; Tab-trap via focus-trap helper; ESC closes; focus returns to opener
```

## AC-4: Sync metrics w/ aria-live + campaign-name error mapping
```gherkin
Given the user clicks "Sync now"
When POST /api/ads/google/campaigns/sync fires
Then during the request, button disables + shows "Syncing…"
And the status message container has role="status" aria-live="polite"
And on 200: router.refresh + "Synced N campaigns."
And on 207: router.refresh + message listing first 2 failures by campaign NAME (via formatSyncErrors helper mapping error.id → campaign.name; fallback to id prefix when name missing)
And on 401/402/403/429/500: container role becomes "alert" with server error
And button returns to idle after response
```

## AC-5: Metrics summary aggregates + tolerates missing/malformed metrics
```gherkin
Given campaigns may have metrics as {}, null, partial fields, or string-coerced numbers
When MetricsSummary renders using readMetrics helper
Then it shows 4 plain cards (Impressions, Clicks, Spend USD, Conversions) — summed across enabled campaigns
And each metric defaults to 0 when the field is missing, null, or non-numeric
And if no campaign has status='enabled', shows "No enabled campaigns — metrics appear after enable + sync."
And formatMicrosAsUsd converts micros to dollars w/ 2-decimal format
```

## AC-6: Client-safe template subset + drift detection
```gherkin
Given server imports CAMPAIGN_TEMPLATES from @/lib/google-ads/templates (server-only)
When client components need picker metadata
Then a client-safe module @/lib/ads/campaign-templates exports CLIENT_CAMPAIGN_TEMPLATES (subset — no full keyword lists)
And NO server-only fields reach the client bundle
And a drift-detection test imports BOTH modules (vitest server-only mock enables this) and asserts the client subset has the same ids + count + name + description as the server full list — any divergence red-fails the test
```

```gherkin
Given pure helpers govern validation + formatting + branching
When tests run

  - validateCampaignCreate → empty name, >255, budget ≤ 0, budget > cap, valid
  - validateBudgetDelta → within 50%, >50%, current=0 special case, negative refuse
  - selectCampaignControls → full 3x3 role/status matrix
  - formatMicrosAsUsd → 0, 1M, 1.5M, 500M, null/undefined/NaN
  - dollarsToMicros → 50 → 50_000_000; 50.50 → 50_500_000; 50.555 → 50_560_000 or rejected (document); non-finite → rejected
  - readMetrics → {}, null, partial, string-coerced, valid — all return safe numbers
  - formatSyncErrors → id → name lookup, fallback, truncation to first 2

```
