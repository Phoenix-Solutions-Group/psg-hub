<objective>
## Goal
Ingest customer reviews from Google Business Profile and Yelp into a multi-tenant reviews table, surface them in the dashboard as a sortable list filtered by shop. Lays the data foundation for 05-02 (AI response drafts).

## Purpose
Phase 5 goal: "BSM covers content, SEO, reputation, and paid search for collision repair." Reputation starts with knowing which reviews exist. Today shop owners have no consolidated view across platforms. This plan consolidates.

## Output
- One migration creating `reviews` and `review_sources` tables with RLS
- Adapter-pattern ingestion library (Yelp Fusion + GBP) with unified shape
- POST ingest endpoint (manual trigger, idempotent on `external_id`)
- GET list endpoint filtered by `shop_id`
- Dashboard `/reviews` route: list view with platform/rating/date columns
</objective>

## AC-1: Reviews schema with RLS
```gherkin
Given migration 003_reviews.sql exists
When applied to Supabase
Then tables `reviews` and `review_sources` exist
And RLS restricts rows to shops the current user belongs to via get_user_shop_ids()
And `(shop_id, platform, external_id)` is unique (dedup key)
```

## AC-2: Yelp + GBP adapters return unified shape
```gherkin
Given YELP_API_KEY and GOOGLE_PLACES_API_KEY env vars set
When adapter.fetch(shop) runs for a shop with platform IDs configured
Then returns Review[] with { platform, external_id, author, rating, body, posted_at, url }
And adapter throws typed error on missing credentials (does not fail silently)
```

## AC-3: Manual ingest endpoint is idempotent
```gherkin
Given a shop with configured review_sources rows
When POST /api/reviews/ingest { shop_id } called twice
Then duplicate reviews are not inserted (ON CONFLICT DO NOTHING on dedup key)
And response returns { inserted, skipped } counts
And endpoint rejects unauthenticated requests (401)
And endpoint rejects users without shop access (403)
```

## AC-4: /reviews page lists reviews per shop
```gherkin
Given a signed-in user with shop membership
When they visit /reviews
Then they see reviews from all their shops
And can filter by shop, platform, rating
And rows show author, rating stars, truncated body, platform, posted_at, external link
And empty state appears when no reviews exist ("No reviews yet. Connect Google or Yelp in settings.")
```

<task type="auto">
  <name>Task 3: Ingest + list API endpoints</name>
  <files>dashboard/src/app/api/reviews/ingest/route.ts, dashboard/src/app/api/reviews/list/route.ts</files>
  <action>
ingest/route.ts — POST handler:
- Auth: createServerClient, check session, 401 if none
- Parse { shop_id } from body, 400 if missing
- Verify membership: query shop_members where profile_id=session.user.id and shop_id=shop_id, 403 if none
- Query review_sources where shop_id and active=true
- For each source: call getAdapter(platform).fetch(source) in parallel (Promise.allSettled)
- Insert reviews via service-role client with ON CONFLICT (shop_id, platform, external_id) DO NOTHING
- Return { inserted: number, skipped: number, errors: [{platform, message}] }

list/route.ts — GET handler:
- Auth required
- Query params: shop_id (optional), platform (optional), min_rating (optional)
- RLS does the scoping — query reviews table directly with filters
- Order by posted_at desc, limit 100
- Return { reviews: Review[] }

Avoid: exposing service-role key to client; catch adapter errors individually so one platform failure doesn't break others.
  </action>
  <verify>curl POST /api/reviews/ingest without session returns 401; with valid session + fake shop returns 403; build passes</verify>
  <done>AC-3 satisfied: idempotent ingest, scoped list</done>
</task>
