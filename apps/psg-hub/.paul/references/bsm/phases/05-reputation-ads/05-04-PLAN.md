## Goal
Ship the Performance-tier billing path and the first slice of the ads dashboard: shop owners can upgrade to Performance ($999/mo) via Stripe Checkout, land on `/ads`, and connect their Google Ads account. Campaigns UI (create, detail, metrics, sync) lands in 05-05.

- `/dashboard/billing` exposes Performance tier card with "Upgrade" CTA + post-upgrade grace banner
- `/ads` page with tier-gated view (tier-gate card for non-Performance; link + list for Performance)
- Link-account button: popup-blocker-safe, snapshot-and-diff polling, postMessage-ready, useEffect cleanup

## AC-5: Link Google Ads account (popup-safe + snapshot polling)
```gherkin
Given a Performance-tier shop with owner role
When the user clicks "Link Google Ads"
Then the client:
  - captures a snapshot Set<customer_id> of existing accounts (via current server-rendered state)
  - POSTs to /api/ads/google/authorize with { shop_id }
  - attempts window.open with the returned url
  - if window.open returns null (popup blocked) → renders inline error "Popup blocked. [Open in new tab]" with anchor fallback, and does NOT start polling
  - on successful popup: starts a 3s-interval poll to /api/ads/google/accounts?shop_id=X
  - also registers window.addEventListener('message') filtered by origin — if a message arrives indicating link success, short-circuits polling
  - stops polling when: (a) polling or postMessage detects a new customer_id NOT in the snapshot with status='linked'; OR (b) popup reports closed (window.closed check); OR (c) 5-minute timeout
  - on success: triggers server component refresh so accounts list re-renders with the new row
  - on timeout: shows "If you completed the link, refresh the page."
  - on unmount: cleanup stops interval + removes message listener (no leaks)
And the button is only rendered when role='owner'; other roles see a read-only note
```

    export type ShopRole = "owner" | "manager" | "viewer";
    export type AdsView = "tier-gate" | "empty-link" | "table" | "upgrade-processing";

    export function selectAdsView(input: {
      tiered: boolean;
      accountsCount: number;
      role: ShopRole;
      justReturnedFromStripe: boolean;
      elapsedMsSinceReturn: number;
    }): AdsView {
      if (!input.tiered) {
        if (input.justReturnedFromStripe && input.elapsedMsSinceReturn < 60_000) {
          return "upgrade-processing";
        }
        return "tier-gate";
      }
      if (input.accountsCount === 0) return "empty-link";
      return "table";
    }
