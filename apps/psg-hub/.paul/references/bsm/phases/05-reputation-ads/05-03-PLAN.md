<objective>
## Goal
Backend integration with Google Ads API for per-shop collision-repair campaign management. Per-shop OAuth with encrypted refresh-token storage (AES-256-GCM), Google Ads API v20 client wrapper, starter collision-repair campaign template library, tier-gated + budget-capped CRUD routes for accounts and campaigns, nightly metrics sync. UI comes in 05-04 — this plan ships the backend surface.

## Purpose
Shops currently pay agencies $2-5K/mo for ad management. BSM undercuts this by running campaigns on their behalf with PSG's collision-repair playbook baked in, while the shop retains ownership of their Google Ads account. Gated behind Performance tier ($999/mo) and hard budget caps to prevent any single shop from burning quota, budget, or PSG's shared developer token.

## Output
- `google_ads_accounts` + `google_ads_campaigns` + `google_ads_oauth_states` + `ads_api_call_log` tables
- Per-shop OAuth 2.0 authorization flow with encrypted refresh-token storage + key-version rotation map
- Google Ads API v20 client wrapper with rate-limit reuse from 05-02's log pattern
- Starter template library (≥3 collision-repair templates)
- 7 API routes (authorize, callback, accounts list, disconnect, campaigns list, campaign detail, metrics sync)
- Tier-gating module + budget-cap enforcement + shop-preflight (address/website/radius required)
- Append-only ads_api_call_log with REVOKE enforcement
- Unit + integration tests (including negative invariants: mock-NOT-called on gated paths)
</objective>

<acceptance_criteria>

## AC-1: OAuth authorize + callback flow
```gherkin
Given an authenticated shop member with owner role
When they POST /api/ads/google/authorize
Then they receive a Google OAuth URL scoped to `https://www.googleapis.com/auth/adwords`, with a signed state parameter tied to user_id+shop_id+nonce (10-min expiry, stored server-side)
And when Google redirects to /api/ads/google/callback with code+state
Then the state is validated (HMAC + expiry + not-replayed), the authenticated user's id matches state.userId (else 403), the code is exchanged for access+refresh tokens, refresh_token is encrypted with AES-256-GCM using the current key version, and a row is INSERTed or UPSERTed (when prior row exists with status in revoked|error) in google_ads_accounts with status='linked'
```

## AC-2: Encrypted refresh-token storage + key rotation
```gherkin
Given a refresh token must be persisted
When stored in google_ads_accounts.encrypted_refresh_token
Then it is AES-256-GCM encrypted (12-byte random IV prepended; auth tag appended)
And the key is loaded from a KEY_VERSION_MAP (v1 from ADS_ENCRYPTION_KEY, v2+ from ADS_ENCRYPTION_KEY_V{N}); encryption uses highest-available version; decryption selects by row.key_version
And decryption fails (throws) on tampered ciphertext or wrong key
And no code path logs the plaintext refresh_token (verified by unit test that stubs console.log/error)
```

## AC-3: Google Ads API client + tenancy
```gherkin
Given a shop has a linked google_ads_account
When any API route calls getGoogleAdsClient(shopId, userId)
Then the helper loads the account, decrypts the refresh token, constructs a google-ads-api v20 Customer client, and attaches shopId+userId to context
And every call is wrapped in withAdsRateLimit(...) + logAdsCall(...) regardless of success/failure
And cross-tenant access returns 403 before any Google API call is issued
```

## AC-4: Campaign template library (collision-repair vertical)
```gherkin
Given campaign templates are bundled with the app
When /api/ads/google/campaigns POST is called with {shop_id, template_id, daily_budget_micros}
Then the selected template provides: name, SEARCH campaign type, collision-repair keyword list (positive + negative), ad group structure, RSA headlines (≤30 chars each, ≤15 total) + descriptions (≤90 chars each, ≤4 total), geo-targeting fields, ad schedule
And a campaign is created via mutate API, stored with external_resource_name, status='paused' (never auto-enabled), template_id reference
And at least 3 templates ship: "storm-damage-response", "insurance-approved-shops", "oem-certified-repair"
And every template passes Google char-limit validation at test time
```

## AC-5: Campaign CRUD + safe mutations
```gherkin
Given campaigns exist in google_ads_campaigns
When /api/ads/google/campaigns GET is called
Then it returns RLS-scoped rows with last-synced metrics (impressions, clicks, cost_micros, conversions; window=last_30d)
And PUT allows budget changes (delta capped at ±50% per 24h) + status changes
And PUT status='paused'→'enabled' (the "first-enable") requires role='owner' (not just owner|manager)
And DELETE sets Google + local status='removed' (never hard-deletes audit record)
And name longer than 255 chars is rejected at 400 pre-mutate
```

## AC-6: Metrics sync endpoint
```gherkin
Given a shop with linked account and campaigns
When POST /api/ads/google/campaigns/sync is called
Then for each active campaign: a GAQL query (with external_id validated as /^\d+$/ before interpolation) pulls metrics.impressions, clicks, cost_micros, conversions for segments.date DURING LAST_30_DAYS
And results are upserted into google_ads_campaigns.metrics jsonb
And partial failures return `{ synced: [...], errors: [...], partial: true }` with status 207 Multi-Status; otherwise 200 with `partial: false`
```

## AC-7: Rate limiting + cost governance (quota)
```gherkin
Given Google Ads API has quota (per-token + per-customer rate limits)
When any /api/ads/google/* route is called
Then the rate-limit layer enforces: `ADS_MUTATE_LIMIT_PER_HOUR` (default 20) mutations per shop per hour, `ADS_READ_LIMIT_PER_HOUR` (default 500) reads per shop per hour
And any 429 from Google upstream is logged + surfaced as 429 with retry-after header
And the rate-limit gate runs BEFORE the Google API call (verified by mock-not-called integration test)
```

## AC-8: OAuth state tamper resistance + auth binding
```gherkin
Given an attacker attempts callback with forged/stale/replayed state
When /api/ads/google/callback validates state
Then HMAC signature mismatch returns 400, expired state returns 400, replay (state.consumed_at not null) returns 400, state not found returns 400
And if the authenticated user at callback does not match state.userId, returns 403
And in every failure case: no database write to google_ads_accounts, no Google token exchange, no upstream call
And expired+consumed states are lazily swept in the same callback (DELETE ... WHERE expires_at < now())
```

## AC-9: Tests assert negative invariants
```gherkin
Given integration tests mock google-ads-api + supabase + Google OAuth endpoints
When cross-tenant, untiered, or rate-limited requests are tested
Then Google Ads mock is asserted NOT called, Google OAuth mock is asserted NOT called
And tampered state is asserted NOT call the token-exchange mock
And the refresh_token plaintext is asserted never present in console log output
```

## AC-10: Tier gating (Performance tier required)
```gherkin
Given a shop either has no active subscription or has an active Essentials/Growth subscription (not Performance)
When any /api/ads/google/* route is called
Then the route returns 402 Payment Required with a message naming the required tier, unless the shop's slug is on the `SHOP_ADS_TIER_OVERRIDE` allowlist env (for internal PSG test shops without billing)
And no Google Ads API call is made, no OAuth flow is initiated, no ads_api_call_log write occurs (the gate precedes logging — tier denial is a billing concern, not an ads-layer concern)
And a shop with an active `performance` subscription tier passes the gate
```

## AC-11: Budget ceiling enforcement
```gherkin
Given a campaign is being created or updated
When daily_budget_micros is submitted
Then the value is rejected with 400 if it exceeds max(shops.max_daily_ad_budget_micros, ADS_MAX_DAILY_MICROS) — default 500_000_000 ($500/day)
And the rejection happens BEFORE the Google mutate call (no quota spent)
And a per-shop override column `shops.max_daily_ad_budget_micros` (nullable bigint) can raise the ceiling for specific shops
```

## AC-12: Shop preflight (address + website + radius)
```gherkin
Given a campaign create is requested
When the shop row is loaded
Then if shop.address is null/empty OR shop.service_radius_miles is null OR shop.website_url is null OR shop.website_url does not start with 'https://'
Then the route returns 400 with specific error identifying missing fields
And no Google mutate call is made
```
