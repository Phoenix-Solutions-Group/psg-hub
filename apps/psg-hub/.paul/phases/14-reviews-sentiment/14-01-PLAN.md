<objective>
## Goal
Ingest per-review Google Business Profile reviews (legacy v4 `accounts.locations.reviews.list`) into the existing `review_items` table, idempotently, via a sync orchestrator + cron that mirror the Phase-13 GBP ingest vertical. Replace the 501 `ingest` stub. Build-local, ZERO prod.

## Purpose
The reply-draft + human-approval pipeline (`responder` / `safety` / draft-response / approve-response) already exists but has no data: the ingest route is a 501 stub and Phase 13 only fetched the lifetime rating AGGREGATE, never per-review rows. This plan fills `review_items` so that pipeline (and Phase-14 sentiment + reply-publish) has reviews to act on. Per 14-RESEARCH: extend, do not rebuild.

## Output
- One additive, idempotent migration: `review_items` gains `external_review_id` + `updated_at` + `UNIQUE(shop_id, external_review_id)`; `analytics_sync_runs` source CHECK widened to admit `'gbp_reviews'`.
- `gbp-review-items.ts` — raw-HTTP per-review v4 fetch (paginated), mirroring the `gbp-reviews.ts` aggregate seam.
- `gbp-reviews-sync.ts` orchestrator + `/api/cron/gbp-reviews-sync` cron (10th) + the `ingest` route rewired to a single-shop ingest.
</objective>

## AC-1: Per-review v4 fetch maps the Review schema into review_items shape
```gherkin
Given a linked gbp account with a non-null externalParentId
When fetchGbpReviews(shopId) issues GET v4/{accounts/{aid}/locations/{lid}}/reviews?pageSize=50&orderBy=updateTime desc
Then it loops nextPageToken to completion and returns rows mapping:
  name -> external_review_id, starRating word-enum -> rating (ONE..FIVE => 1..5, STAR_RATING_UNSPECIFIED => null),
  comment -> text, reviewer.displayName -> author (null when reviewer.isAnonymous), createTime -> reviewed_at (ISO), updateTime -> updated_at
```

## AC-2: Unverified / empty location degrades, never trips the breaker
```gherkin
Given a non-VoM / unverified location (a non-2xx reject OR a 200 with reviews absent)
When fetchGbpReviews runs
Then it returns an empty array (no fabricated rows), the orchestrator writes zero rows for that shop, the shop is counted skipped, and the account is NOT flipped to status='error'
```

## AC-3: Orchestrator ingests idempotently with contained per-shop failure
```gherkin
Given multiple shops holding a status='linked' gbp google_oauth_accounts row
When syncGbpReviews(service) runs and re-runs
Then each shop's internal primary location_id is resolved, rows upsert into review_items onConflict (shop_id, external_review_id), a re-run nets zero new rows,
  a single shop's failure is contained (auth_failed flips that account via markAccountError), and one analytics_sync_runs ledger row is written with source='gbp_reviews'
```

## AC-4: The 501 ingest stub is replaced by a working single-shop ingest + cron
```gherkin

```
