<acceptance_criteria>

## AC-1: Per-shop analytics surface
```gherkin
Given an authenticated customer with an active shop that has semrush snapshots
When they visit /dashboard/analytics
Then they see KPI stat cards + time-series charts rendered from that shop's snapshots only,
  with a "Last synced" timestamp derived from the newest row
And "Analytics" appears in the dashboard NAV (sidebar + mobile)
```

## AC-2: Designed empty state
```gherkin
Given an active shop with zero snapshot rows (prod reality until 09-03 ships)
When the user visits /dashboard/analytics
Then a designed empty state renders (brand card, explains data syncs on a cadence)
  — no crash, no blank chart shells, no raw "0" KPI lies
```

## AC-3: MSO cross-shop aggregate
```gherkin
Given a user with 2+ shop memberships
When they switch the analytics scope to "All shops"
Then metrics are aggregated (summed per date) across ALL their authorized shops
And a single-shop user never sees the scope control
And the aggregate read is clamped to their membership set (explicit shop_id IN list + RLS backstop)
```

## AC-4: Switcher typeahead (big MSO)
```gherkin
Given a user with 8 or more shop memberships (E2E fixture: MEGA, 9 shops)
When they type in the switcher search input
Then the visible shop option count reduces to the matches (case-insensitive)
  — verified LIVE in Playwright, not just at the pure-function level
Given a user with 2-7 shops
Then the existing plain select renders unchanged (shop-switch.spec.ts passes unmodified)
```

## AC-5: LCP regression gate
```gherkin
Given the prod-build E2E harness (local Supabase, storageState auth, CPU throttled 4x)
When e2e/lcp.spec.ts measures /dashboard and /dashboard/analytics
Then /dashboard median LCP of 4 samples (warm-up discarded) is < 2000ms (HARD gate — the ROADMAP metric route)
And /dashboard/analytics median is logged + asserted under a 4000ms regression ceiling
  (the new recharts-heavy route gets its hard budget calibrated at 09-03 when real data lands;
   the throttled-localhost number is a regression ceiling, NOT a field-LCP predictor —
   field <2s p75 is a separate post-deploy CrUX/RUM check, out of this spec's scope)
```

## AC-6: Real chart render + axe AA
```gherkin
Given seeded snapshot fixtures in the local E2E stack
When Playwright loads /dashboard/analytics in a real browser
Then recharts SVG geometry is present (path elements rendered)
And axe WCAG2a/2aa reports 0 serious/critical violations on both per-shop and aggregate views
```
