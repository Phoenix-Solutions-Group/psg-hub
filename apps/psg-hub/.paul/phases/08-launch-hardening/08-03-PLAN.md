<objective>
## Goal
Run the first AEGIS diagnostic pass on psg-hub, scoped to the customer-facing surface and the launch-relevant domains (security, compliance, correctness, data), then triage and remediate the launch-blocking findings before a real shop logs in with live PII. Defer the full 14-domain enterprise audit to the v2.0 final AEGIS.

## Purpose
This is the M2 "first AEGIS pass (pulled forward from v2.0)" readiness gate and the last hardening step in Phase 8 before pilot onboarding onto live data. Phase 8 has already closed the RLS cross-tenant breach (08-02) and the secret scan (08-02b); AEGIS is the structured adversarial sweep that catches what those targeted passes did not — auth/session handling, input validation, data-exposure, error handling, dependency risk across the customer surface. The pilot cohort (Wallace/Tedesco/Tracy's) cannot safely go live until the customer-facing surface passes a real security/correctness audit.

## Output
- An initialized `.aegis/` project + a completed scoped Core audit (Phase 0-5): threat model, domain findings (security/compliance/correctness/data), and a ranked report under `.aegis/report/`.
- A triaged finding inventory split into remediate-now (launch-blocking) vs defer (later milestone / v2.0 final AEGIS), recorded to STATE `### Decisions`.
- Remediations applied for the accepted set, re-gated (typecheck/test/build green) and re-validated; any DB change via migrations-as-code under the 06-01 PROTOCOL.
- Member flows on hub.psgweb.me verified unbroken.
</objective>
