<objective>
## Goal
Make psg-hub onboardable: a logged-in user with no shop completes the onboarding wizard and ends up with a real `shops` row (live columns) + a first-owner `shop_users` row + a `customer` `app_user_roles` row, created server-side via the service-role client — the bootstrap that the Phase 6 spine made impossible from the browser.

## Purpose
Phase 6 unified membership on `shop_users` but its INSERT `with_check = user_is_shop_owner(shop_id)` is chicken-and-egg for a brand-new shop, so the first owner cannot be created under the browser/user-session client. The wizard's direct insert is therefore dead, and the system is un-onboardable except by manual SQL. This is the prerequisite for the rest of v0.2: the shop switcher (07-03) has nothing to switch between, and tier gating (07-02) keys on a shop's subscription — both assume a shop exists. It also reconciles the wizard's phantom `shops` column writes (live `shops` has no `website_url`/`phone`/`city`/`state`/`address`) found at 06-05 human-verify.

## Output
- `POST /api/onboarding` — service-role route: validates the caller, creates the shop (real columns), first-owner `shop_users`, customer `app_user_roles` (no clobber of existing role), compensating cleanup on partial failure.
- `onboarding-wizard.tsx` calls the route (no direct browser `shops`/`shop_users` insert).
- Route unit tests + green gates.
- `07-01-SUMMARY.md`.
</objective>
