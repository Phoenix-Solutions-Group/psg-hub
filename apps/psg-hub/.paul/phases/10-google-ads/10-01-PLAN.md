<objective>
## Goal
Provision the four absent Google Ads tables (`google_ads_accounts`, `google_ads_campaigns`, `google_ads_oauth_states`, `ads_api_call_log`) as a migration-as-code with per-table RLS, LOCAL-applied; then PROVE the already-built (blind, against a schema that existed nowhere) OAuth-link + accounts-read code runs against the real schema, and bring the `/dashboard/ads` accounts/link surface online in its unlinked state.

## Purpose
Phase 10's entry condition (re-plan grounding 2026-06-04): the Google Ads OAuth + client + campaign code and the full `/dashboard/ads` UI are built, but every table they read/write is absent on prod AND in every migration AND in the `remote_schema` dump. Nothing can run until the tables exist. This plan makes the schema real and de-risks the blind-built code with a real-client check (the 09-02 pattern — a real INSERT, not a mocked unit test, is what catches column/constraint drift). It unblocks 10-02 (analytics-shell ingest) and the operator activation gate batch.

## Output
- One idempotent migration creating 4 tables + indexes + RLS (LOCAL-applied only; prod apply is the Phase-10 gate batch).
- `/dashboard/ads` flipped from the "coming soon" guard to the real accounts surface (unlinked/empty + link CTA state, since no account is linked locally).
- A real-client schema integration check + an E2E google-ads spec proving the migration matches the built read/link/log code under RLS.
- `10-01-SUMMARY.md`.
</objective>

<decisions>
## Resolved at plan time (2026-06-08, operator)
- **Token encryption = app-level AES-256-GCM (the built `src/lib/google-ads/crypto.ts`), NOT pgsodium.** Column shape: `encrypted_refresh_token bytea` + `key_version integer`. This is a **recorded deviation** from PROJECT.md's "pgsodium encryption at rest for OAuth refresh tokens" constraint: the inherited code is built + unit-tested with versioned app-key GCM (genuine encryption-at-rest), and re-doing it as pgsodium is pure risk for no security gain. **Phase 11 (GA4+GSC) inherits this choice** for refresh-token consistency; the ROADMAP's "pgsodium-encrypted refresh tokens" note for Phase 11 reconciles to app-key GCM. UNIFY logs this to PROJECT decisions.
</decisions>
