## Goal
Build the Google Ads daily ingest end to end, locally, with ZERO prod contact: an
account-level GAQL daily-totals query, a `syncGoogleAdsSnapshots` orchestrator mirroring
the 09-03 SEMrush path (idempotent `analytics_snapshots` rows, `source='google_ads'`,
`period='daily'`), a `CRON_SECRET`-gated cron route, and a paid panel on
`/dashboard/analytics`. First-live-run contract verification is explicitly a 10-03 gate item,
not this plan's done-state.

## Purpose
Phase 10's core value: paid-marketing metrics (spend, clicks, conversions, CPL) in the same
unified analytics surface as organic SEO. 10-01 provisioned the `google_ads_*` tables + the
unlinked/link surface; 10-02 turns a linked account into time-series snapshots and surfaces
them. The ingest pattern (client → orchestrator → ledger → idempotent upsert) is the 09-03
template; this plan reuses it for the second source and proves the surface is genuinely
source-agnostic.

## AC-1: Account-level daily metrics query (contract-anchored, not blind-mocked)
```gherkin
Given a shop with a linked Google Ads account and a settled date string "YYYY-MM-DD"
When fetchAccountDailyMetrics(shopId, dateStr) runs
Then it issues ONE GAQL query `SELECT metrics.cost_micros, metrics.clicks, metrics.impressions,
     metrics.conversions FROM customer WHERE segments.date BETWEEN '<d>' AND '<d>'` (FROM customer,
     BETWEEN — never `segments.date = 'd'`, which RESEARCH adversarially REFUTED, and never a
     segment in the SELECT)
And it returns GoogleAdsMetrics { spend = cost_micros/1_000_000, clicks, impressions, conversions,
     cpl = conversions>0 ? spend/conversions : null, cost_micros }
And a zero-activity day (ZERO returned rows, not a row of zeros) yields all-zero metrics with cpl=null
And unit fixtures are shaped to the EXACT google-ads-api@23 parserRest output (snake_case keys,
     INT64 cost_micros/clicks/impressions already coerced to Number) — cited in a test comment —
     so the test anchors to the library contract, not a hand-invented shape
And the query is wrapped in a module-level CircuitBreaker + withRetry (transient-only; auth_failed/
     bad_request never retried), inside the existing withAdsRateLimit guard — satisfying the PROJECT.md
     "retry + circuit breaker on every external call" constraint the inherited path was missing.
```

## AC-2: Per-shop-contained ingest orchestrator (mirrors 09-03)
```gherkin
Given multiple shops, only some holding a google_ads_accounts row with status='linked'
When syncGoogleAdsSnapshots(service, options) runs
Then it ingests ONLY status='linked' shops (status='error'/'revoked' and unlinked shops are skipped)
And for each eligible shop it upserts one analytics_snapshots row per target date
     (source='google_ads', period='daily') over the idempotency key — same-day re-runs net zero new rows
And a single shop's AdsApiError('auth_failed') is contained: that shop counts failed, the batch
     continues, AND markAccountAuthFailed flips its account to status='error' so the 10-01 accounts
     surface shows "needs re-link"
And one analytics_sync_runs ledger row (source='google_ads') opens 'running' and closes
     'success'/'error' with rows_written — a ledger failure never masks the sync result.
```

## AC-3: CRON_SECRET-gated trigger, zero unauthorized spend
```gherkin
Given the /api/cron/google-ads-sync route
When it is called without a valid `Authorization: Bearer ${CRON_SECRET}` (incl. CRON_SECRET unset)
Then it returns 401 BEFORE constructing any client or reading any shop (zero Google API spend)
And when authorized but server Google creds are absent it returns a designed not-configured 503
And when authorized + configured it runs the sync and returns the SyncResult counts
And vercel.json declares a 2nd daily cron for this path.
```
