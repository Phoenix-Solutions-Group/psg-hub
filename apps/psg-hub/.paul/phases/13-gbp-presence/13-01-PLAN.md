<objective>
## Goal
Build the GBP OAuth link foundation for Phase 13: a SEPARATE "Connect Google Business Profile" consent
(Option B — research §Recommended architecture #1) carrying the `business.manage` scope, with its own
`/api/analytics/google/gbp/{authorize,callback,select}` routes and its own `google_oauth_accounts` row
(`source='gbp'`). The callback enumerates the user's GBP accounts (My Business Account Management
`accounts.list`) and their locations (My Business Business Information `accounts.locations.list`), the
operator picks one location, and it persists as ONE row storing the bare `locations/{id}` as
`external_account_id` PLUS the parent `accounts/{id}` alongside. No insights fetch, no panel, no cron —
those are 13-02 (daily insights) and 13-03 (monthly presence), which both depend on this account model.

## Purpose
13-02 (daily GBP insights) and 13-03 (monthly presence + star rating) cannot run without a stored,
per-shop refresh token carrying `business.manage` and a selected `locations/{id}`. The Phase-11 combined
consent grants ONLY `analytics.readonly + webmasters.readonly` (verified in `authorize/route.ts:12-13`),
so every GBP call 403s and every linked shop — including the Wallace pilot — must re-consent under the
widened scope. A separate flow keeps the GA4/GSC token's blast radius small (a broken GBP re-consent can
never invalidate a working GA4/GSC token). This extends the v0.3 analytics surface toward GBP presence.

## Output
- 1 NEW migration (LOCAL-applied only): widen `google_oauth_accounts` source CHECK to admit `'gbp'`, ADD
  a nullable `external_parent_id` column (holds `accounts/{id}` for gbp; null for ga4/gsc).
- A NEW `src/lib/google-oauth/gbp-enumerate.ts` (account flatten + location list, the `googleapis` `auth:`
  injection idiom — NOT gax) + the 3 GBP link routes + a link button.
- `GoogleOAuthSource` union widened to `'ga4' | 'gsc' | 'gbp'`; `persistLinkedAccount` carries the parent id.
- Unit tests for enumeration; an e2e schema round-trip proving the `source='gbp'` row + parent id + RLS.
- At UNIFY: 13-01-SUMMARY. LIVE activation (real GBP consent, deploy) is the Phase-13 gate batch (13-04),
  itself behind the two Google lead-time gates — which the operator MUST start in parallel TODAY (below).
