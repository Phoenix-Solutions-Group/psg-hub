<acceptance_criteria>

## AC-1: Resilient SMS send
```gherkin
Given the Twilio SMS adapter wraps every send in the shared CircuitBreaker.execute(withRetry(...))
When a send hits a transient failure (HTTP 429 / 5xx / network error with no HTTP status)
Then it retries with bounded exponential backoff, and after repeated transient failures the breaker opens (CircuitOpenError surfaces as a retryable SmsError)
And When a send hits a permanent failure (a non-429 4xx, e.g. Twilio code 21211 invalid number)
Then it does NOT retry and does NOT trip the breaker, surfacing a non-retryable SmsError
```

## AC-2: Idempotent, signature-verified, dual-path webhook
```gherkin
Given POST /api/webhooks/twilio receives a Twilio request (application/x-www-form-urlencoded)
When the X-Twilio-Signature is valid (HMAC-SHA1 over the env-reconstructed public URL + parsed POST params, keyed by TWILIO_AUTH_TOKEN)
Then the event persists to sms_events and the route returns the correct ack (204 for a status callback; empty TwiML with Content-Type text/xml for an inbound message), only after a successful persist
And When the same (message_sid, status) pair is delivered again (Twilio retry)
Then upsert(onConflict "message_sid,status", ignoreDuplicates) makes it a no-op — no duplicate row, distinct lifecycle states (queued/sent/delivered) still persist as separate rows
And When the signature is missing → 400; invalid → 403; TWILIO_AUTH_TOKEN or TWILIO_WEBHOOK_BASE_URL unset → 500 (fail closed); the Supabase persist fails → 500 (so Twilio retries, idempotently)
```

## AC-3: Operator provisioning + live send (send-half verifiable NOW)
```gherkin
Given the operator has provisioned Twilio Account SID + Auth Token + a Messaging Service SID containing an SMS-capable number (destination verified if the account is in trial) into .env.local
When `node --env-file=.env.local scripts/send-test-sms.mjs` runs a real messages.create()
Then the call returns a non-error status (queued | accepted | sent — NOT "delivered") and the destination phone physically receives the SMS
# Live webhook signature verification (status-callback delivered-row + inbound) is DEFERRED to 03-04,
# which provides the stable public URL Twilio must sign against — exact parallel to 03-01's deferred webhook-row verify.
```
