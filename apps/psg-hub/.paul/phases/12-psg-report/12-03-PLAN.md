<objective>
## Goal
Build the report ARTIFACT vertical: a branded, internal, token-gated print route that renders `ReportData` (12-01) + the eval-passed `ReportNarrative` (12-02) into the canon HTML/CSS; a render client that drives a controlled-host (Hetzner) Chromium worker over HTTP to turn that route into a PDF; private Supabase Storage persistence; a session + membership-gated customer download route; and a pure link-email payload builder. All build-local, deps-injected, ZERO prod contact; the `monthly_reports` migration is authored but NOT applied.

## Purpose
This is the deliverable a shop owner actually receives. RESEARCH pins the engine (headless Chromium `page.pdf()` over the existing branded HTML — the only path that reuses the canon's `@media print`/`@page`/bar-fill CSS verbatim) and the runtime (Vercel Fluid Compute REFUTED for Chromium — `libnss3.so` launch break; render on a host PSG controls). 12-03 builds every local half of that pipeline so 12-04's cron only has to schedule it and the operator gate batch only has to activate it (apply migration, create the bucket, deploy the worker, set secrets + the SendGrid template, run one live render smoke). Same build-local -> operator-gate pattern proven in Phases 9, 10, 11.

## Output
- `report/render.ts` (pure HTML), `app/reports/[slug]/print/route.ts` (token-gated print surface), `report/render-client.ts` (worker HTTP client), `report/storage.ts` (private bucket), `workers/report-renderer/` (Hetzner puppeteer worker, deploy-gated), the `monthly_reports` migration (authored), `app/api/reports/[shopId]/[period]/download/route.ts` (membership-gated), `report/email.ts` (link-email builder) + tests.
- The app adds NO new runtime dependency (render-client uses fetch; storage uses the installed `@supabase/supabase-js`). `puppeteer` lives ONLY in the worker's own `package.json`.
</objective>

## AC-1: A branded, internal, token-gated print route renders the report from ReportData + the verified narrative
```gherkin
Given a ReportData payload (12-01) and an eval-passed ReportNarrative (12-02) for a shop+month
When GET /reports/{slug}/print is requested with a valid RENDER_TOKEN bearer
Then render.ts returns ONE branded HTML document in the canon layout: masthead (eyebrow + h1 + .sub + .meta window/sources/status) -> "story so far" 4 KPI cards (GA4 sessions / GSC clicks / Ads conversions / SEMrush organic_keywords, each with its MoM .chg) + .callout (narrative.executiveSummary) -> per-source sections for each linkedSource (GSC, GA4, Ads, SEMrush) with the canon trend-bar + table.psg patterns -> "this month vs prior" MoM table.psg -> "what is driving movement" .takeaways (narrative) -> recommendations (narrative.recommendations) -> sources/method -> footer "Prepared by Phoenix Solutions Group ... {date}"
And the page uses the app's next/font (gotham + didact from src/lib/fonts.ts) so fonts self-serve over the origin and the Hetzner worker's page.goto resolves them (NOT the canon's relative url("fonts/...") @font-face)
And every numeral in the rendered HTML comes from ReportData / the substituted narrative (no invented numbers); sources NOT in linkedSources are absent (never zero-filled); a source NOT in sourcesWithPriorMonth renders within-period (cold-start) framing instead of a MoM delta
And a request without a valid RENDER_TOKEN returns 401 (the print route is internal — customers never hit it)
```
