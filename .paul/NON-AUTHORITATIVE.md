# `.paul/` is NON-AUTHORITATIVE

PAUL is **contained**: kept project-local to `psg-hub` only, declared non-authoritative,
with **no global / company-wide install**.

Everything under `.paul/` (and `psg-hub/PLANNING.md`) is a draft/input artifact, **not a
system of record**. **Paperclip is the single system of record** for scope, status,
priority, and ownership. Where any `.paul/` artifact disagrees with Paperclip, Paperclip
wins.

To act on PAUL planning content, convert it into Paperclip issues
(`paperclip-converting-plans-to-tasks`) — the Paperclip issue is authoritative.

Decision record: `docs/governance/track-c-framework-containment.md` (PSG-346 · Track C PSG-328).
